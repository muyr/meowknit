# MeowKnit 双语网站设计

## 背景

MeowKnit 需要在现有静态作品集 MVP 上支持简体中文和英语。当前产品内容已经集中到 `src/content/`，图片放在 `public/images/products/`，适合在不引入后台的前提下做轻量多语言。

本设计只覆盖公开网站的双语展示，不引入 CMS、自研后台或复杂翻译工作流。

## 语言范围

首版支持两种语言：

- 简体中文：`zh-CN`
- 英语：`en`

默认语言为简体中文。

## URL 策略

采用路径区分语言：

- `/zh-CN` 显示简体中文。
- `/en` 显示英语。
- `/` 默认进入简体中文体验。

这个方案适合品牌站，因为每个语言版本都有独立 URL，方便分享、调试、SEO 和未来添加 `hreflang`。

## 用户体验

页面应提供清晰的语言切换入口：

- 当前语言为中文时，可切换到 English。
- 当前语言为英语时，可切换到 简体中文。
- 切换语言时保留当前浏览上下文的基本体验，例如仍在作品列表页，已选择的分类可以尽量保留。

首版不需要根据浏览器语言自动切换，以避免用户打开链接后语言不可预期。未来可以在 `/` 上增加浏览器语言检测，但具体语言路径仍应保留。

## 内容模型

产品、分类和 UI 文案都应支持双语。

### 产品内容

产品仍维护在 `src/content/products.ts`，但可翻译字段应改成按语言存储：

- 产品名称。
- 产品描述。
- 图片 `alt` 文案。

不需要翻译的字段保持共享：

- `slug`
- `category`
- `featured`
- `etsyUrl`
- `inquiryUrl`
- `socialUrl`
- 图片 `src`
- 图片 `gradient`

### 分类内容

分类仍维护在 `src/content/categories.ts`，但分类名称和描述应支持双语。

分类 id 继续保持稳定英文 id，例如 `knitwear`、`crochet-bag`，用于筛选和数据关联，不参与翻译。

### UI 文案

固定 UI 文案应放到 `src/i18n/` 中统一管理，包括：

- Hero 区域文案。
- 分类筛选中的“全部作品”。
- 作品区标题。
- CTA 文案，例如 Etsy、咨询、Instagram。
- 页脚文案。
- 语言切换按钮文案。
- 关闭详情按钮文案。

## 技术方向

不引入大型 i18n 依赖，先实现轻量本地方案：

- 新增 `src/i18n/locales.ts`：定义语言类型、默认语言、可用语言和 UI 文案。
- 新增 `src/i18n/routing.ts`：从路径读取语言、生成切换路径、处理默认路径。
- 更新 `src/content/catalog.ts`：根据当前语言返回可渲染的产品、分类和 CTA。
- 更新 Vue 组件：通过 props 接收已翻译内容和 UI 文案，避免组件内部硬编码中文。

保持当前单页静态网站形态，适配 Vite 和 Vercel 静态部署。

## 路由和状态

首版可以不引入 Vue Router，直接使用浏览器路径：

- 启动时从 `window.location.pathname` 读取语言。
- 如果路径不是 `/zh-CN` 或 `/en`，默认使用 `zh-CN`。
- 切换语言时使用 `history.pushState` 或 `location.href` 更新路径。

如果后续需要作品详情独立 URL，再评估是否引入 Vue Router。

## 测试要求

需要覆盖：

- 默认路径显示中文内容。
- `/en` 显示英文 Hero、分类、产品名和产品描述。
- 语言切换按钮能切换到另一种语言。
- 产品 CTA 在不同语言下显示对应文案，但链接保持不变。
- 分类筛选在双语内容下仍然有效。
- 每个产品都有中文和英文内容，避免缺译。

## 非目标

首版不做：

- 自研翻译后台。
- 自动机器翻译。
- 浏览器语言自动跳转。
- 多语言 SEO 元标签和站点地图。
- 每个作品详情的独立多语言 URL。
- 第三种语言。

## 成功标准

双语功能完成后：

- 访客可以通过 `/zh-CN` 和 `/en` 访问对应语言版本。
- 页面中主要品牌文案、分类、产品名称、产品描述和 CTA 都能切换语言。
- 内容维护者可以在 `src/content/` 中清楚地维护双语产品内容。
- 测试和构建通过。
