# MeowKnit 双分类筛选设计

## 背景

当前作品集只有一套分类字段 `category`，无法同时表达“作品用途”和“手工工艺”。MeowKnit 需要支持两种独立分类：

- 用途分类：作品面向的使用场景，例如家居、首饰、宠物用品。
- 工艺分类：作品使用的手工技法，例如梭编、钩针、绳编。

本设计面向现有静态 Vue 作品集，不引入后台、数据库或大型筛选框架。

## 目标

- 每个产品同时拥有一个用途分类和一个工艺分类。
- 作品集页显示两组筛选器：用途筛选和工艺筛选。
- 两组筛选器可以组合使用，例如“家居 + 钩针”。
- 产品卡片和详情页展示用途标签与工艺标签。
- 分类名称和说明继续支持简体中文与英语。
- 内容维护者可以在 `src/content/` 中清楚维护两套分类和产品归属。

## 数据模型

现有 `WorkCategoryId` 应拆分为两类 id：

- `UsageCategoryId`：用途分类 id。
- `CraftCategoryId`：工艺分类 id。

产品数据从：

```ts
category: 'home-decor'
```

改为：

```ts
usageCategory: 'home'
craftCategory: 'crochet'
```

分类文件建议拆成：

- `src/content/usageCategories.ts`
- `src/content/craftCategories.ts`

每个分类保留双语字段：

- `label`
- `description`

## 首版分类集合

用途分类首版建议：

- 家居 / Home
- 首饰 / Jewelry
- 宠物用品 / Pet Goods
- 包袋 / Bags
- 礼物 / Gifts

工艺分类首版建议：

- 梭编 / Tatting
- 钩针 / Crochet
- 绳编 / Macrame
- 针织 / Knitting

分类数量可以后续调整，但 MVP 阶段应避免分类过细。

## 筛选行为

作品集页显示两个筛选组：

- 用途：全部用途、家居、首饰、宠物用品、包袋、礼物。
- 工艺：全部工艺、梭编、钩针、绳编、针织。

筛选逻辑：

- 如果用途为“全部用途”，不限制用途。
- 如果工艺为“全部工艺”，不限制工艺。
- 如果两者都选择具体值，产品必须同时满足两个条件才显示。
- 切换任一筛选项时，关闭当前打开的作品详情。
- 切换语言时保留当前页面，筛选状态可以保留。

## UI 展示

作品集页中，两个筛选组应有明确标题：

- 用途
- 工艺

作品卡片中展示两个标签：

- 用途标签。
- 工艺标签。

作品详情页也展示两个标签，帮助访客理解作品属性。

## 双语要求

以下内容必须支持 `zh-CN` 和 `en`：

- 用途分类名称和说明。
- 工艺分类名称和说明。
- “全部用途”“全部工艺”“用途”“工艺”等固定 UI 文案。

产品名称、描述、图片 alt 继续沿用现有双语内容结构。

## 测试要求

需要覆盖：

- 每个产品都有合法的 `usageCategory` 和 `craftCategory`。
- 每个用途分类和工艺分类都有中英文内容。
- 作品集页显示两组筛选器。
- 单独选择用途可以过滤作品。
- 单独选择工艺可以过滤作品。
- 同时选择用途和工艺时执行组合筛选。
- 卡片或详情页展示用途标签与工艺标签。
- 中英文下筛选器文案都正确显示。

## 非目标

首版不做：

- 多选筛选，例如同时选择多个用途。
- URL query 参数同步筛选状态。
- 后台编辑分类。
- 分类图标。
- 自动根据产品图片或描述推断分类。

## 成功标准

完成后：

- 访客可以从用途和工艺两个维度浏览作品。
- 内容维护者能在 `src/content/` 中清楚维护分类和产品归属。
- 双语页面中分类筛选、标签和产品展示都正常。
- 测试和构建通过。
