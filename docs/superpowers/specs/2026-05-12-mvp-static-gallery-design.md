# MeowKnit MVP 静态作品集设计

## 背景

MeowKnit 是一个个人手工品牌网站。MVP 需要验证：一个精致的线上作品集，是否能有效传达品牌气质、展示手工作品，并引导感兴趣的访客前往 Etsy、发起咨询或关注社交账号。

MVP 使用最简单可行的内容模型：作品信息和图片由技术维护者在结构化源码文件中维护，并提交到 GitHub。Vercel 从仓库自动部署网站。

## MVP 决策

采用方案 A：基于 Vue 的静态作品集网站，使用本地结构化数据和图片资源。

选择这个方案作为 MVP 的原因：

- 当前首要目标是品牌和作品展示，而不是内容运营系统。
- 初始作品规模为中等，大约 15-50 个作品。
- 预计每月更新几次，因此第一版可以接受由技术人员手动维护内容。
- 在展示价值被验证之前，自研管理员后台会过早引入认证、上传存储、校验和权限等复杂度。

## 目标

- 将 MeowKnit 呈现为温暖、时尚、具有明确手工质感的品牌。
- 让访客可以按作品类型浏览作品。
- 通过高质量图片和简短描述，让每个作品显得被认真呈现。
- 支持混合行动入口：有 Etsy 链接时引导到 Etsy；没有链接时引导咨询或关注。
- 保持实现方式兼容 GitHub 和 Vercel 自动部署。

## 非目标

- MVP 不做自研管理员后台。
- 不做用户登录、角色或权限。
- 不做自助图片上传流程。
- 不做购物车、结账、库存或支付。
- 不做复杂搜索。
- MVP 不做多语言内容。
- 不做博客、长故事系统、评论、收藏或客户账号。

## 受众

主要访客是发现这个手工品牌的人。他们会浏览过往或当前作品，并决定是否关注、咨询或访问 Etsy。

MVP 阶段的内部维护者是技术维护者，他们可以更新纳入版本控制的数据和图片。

## 信息架构

MVP 应包含：

- 首页：品牌介绍、视觉氛围、精选作品，以及进入作品集的入口。
- 作品列表页：用响应式网格展示全部作品。
- 分类浏览：使用 5-6 个作品类型分类作为主要浏览方式。
- 作品详情页：聚焦展示单个作品。
- 轻量关于/联系区域：提供足够的信任信息，以及清晰的咨询或关注方式。

## 分类模型

主分类采用作品类型。最终分类名称可以在录入内容时确定，但 MVP 应假设有 5-6 个可见分类。

避免创建很多只有一两个作品的分类。如果某个作品类型内容太少，应先使用更宽泛的分类，等作品数量增长后再细分。

未来版本可以增加二级标签，用于表示系列、季节、可购买状态或定制状态。

## 作品详情内容

每个作品详情页应保持轻量：

- 名称。
- 作品类型分类。
- 一张或多张图片。
- 简短描述。
- 可选 Etsy 链接。
- 可选咨询或社交关注入口。

MVP 不要求长篇故事、规格、尺寸、材料、价格或库存状态。数据模型可以为这些字段预留空间，但第一版 UI 不应依赖它们。

## 行动入口

MVP 应使用混合行动入口：

- 如果作品有 Etsy 链接，显示主要 Etsy 入口。
- 如果作品没有 Etsy 链接，显示咨询或关注入口。
- 全站级行动入口可以指向 Etsy 店铺、Instagram 和联系渠道，具体取决于可用资源。

整体语气应像手工品牌的邀请，而不是强硬的电商结账漏斗。

## 内容管理

MVP 阶段，内容通过源码管理：

- 作品元数据存放在本地结构化数据文件中。
- 图片存放在仓库中，或其他与 Vercel 兼容的简单静态资源位置。
- 更新流程是编辑数据和图片资源，提交到 GitHub，然后由 Vercel 自动部署。

数据结构应足够清晰，方便未来迁移到 Git-based CMS 或自研管理员系统，而不需要重做公开网站 UI。

## 技术方向

- 前端使用 Vue。
- Python 可在未来用于工具脚本或后端需求，但 MVP 除非出现明确需求，否则应避免引入后端。
- 网站应能从 GitHub 部署到 Vercel。
- 在合适情况下优先使用静态生成或静态友好的路由方式。
- 依赖应保持克制，并服务于作品集/图库网站的需求。

## 设计方向

视觉风格应体现：

- 时尚。
- 温暖。
- 手工感和个人品牌感。
- 以图片为主导。
- 精致，但不要显得像企业官网。

UI 应给图片留出呼吸感，使用克制的排版，并让分类易于扫读。

## 成功标准

如果满足以下条件，MVP 可视为成功：

- 访客能从首页快速理解品牌。
- 访客能顺畅浏览 15-50 个作品。
- 作品类型分类能让作品集更容易探索。
- 访客能打开一个作品，并看到图片、简短描述和合适的行动入口。
- 技术维护者可以通过修改结构化数据和图片来新增或更新作品。
- 网站可以通过连接 GitHub 的 Vercel 自动部署。

## 延后升级路径

当 MVP 证明公开展示价值后，再评估：

- 为无技术背景成员提供内容编辑能力的 Git-based CMS。
- 仅当 CMS 方案无法满足团队工作流时，再考虑自研管理员后台。
- 用于系列、季节、可购买状态和定制作品的二级标签。
- 更丰富的作品详情字段，例如材料、尺寸、价格说明和制作故事。
- 更多转化路径，例如邮件订阅或活动落地页。
